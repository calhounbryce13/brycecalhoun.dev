Frontend Design

Frontend design is a process that's implemented in order to facilitate a positive experience for a user of a web service or application. Frontend design emphasizes crafting an efficient user experience. This process includes both the Graphical User Interface (GUI) as well as the manner in which the user interacts with the software. A consistent color and font scheme is crucial to frontend design. Additionally, navigational systems that make sense in the context of the application are just as imperative.

The Five E's of Frontend Design:

Effective
The application needs to meet the goal(s) of the user.
Efficient
Allowing the user to complete tasks with the least number of steps possible.
Easy to navigate
Users should know immediately how to locate and navigate to their goal/objective
Engaging
Entices the user to return by appealing to the unique needs of the intended audience.
Error-free
Trying to anticipate and prevent common errors that users may run into.

The header tag marks the banner or headline for the application and typically stays the same between each webpage. The nav tag acts as a navigation block and can be used various times throughout a webpage. Navigation blocks store links to other webpages or sites. The main tag is simply a layout tag that signifies the primary block of content for that page. The section tag is used to group together content thematically. Typically, a headline tag will follow to describe that section. The article tag is mostly used within a section tag and often comes with a second level headline tag. The aside tag will include content that is related to an article. This information is often seen floating off of the side of text. The figure/figcaption tags are similar to the aside tag, except they are used mostly for media instead of text. The blockquote tag is reserved for quotes that are longer than paragraph length. The footer tag is used below the main tag and will usually hold legal content or contact information. Finally, the div tag is used as a placeholder for dynamic content or simply to visually divide sections.

Ordered Lists:

  1. Ordered lists link to external pages using anchors to those pages
  2. Ordered lists link to internal pages using anchors to those pages
  3. Ordered lists link from page to page by using anchors to those pages

Optimizing Images

There are 6 major specifications for web images to ensure that quality and appropriate photos are uploaded to web pages. The first is a descriptive filename, this allows improved search engine optimization(SEO). Another would be to ensure a small file size so that the load time for each image is as fast as possible. Next, the exact dimensions of the image are needed so that the image doesn't take up any extra space or look too small. The correct file format is also required in order to optimize the graphics of the image. Reduced resolution is also important to make sure that different monitors load the image based on the screen resolution. Finally, color mode allows for the true colors of the image to show.

There are various file formats that are appropriate for media, depending on the type of image. For photography and images .jpg or WebP work the best whereas for line art .gif and .png work better.

Favicons

Favicon stands for favorite's icon, it is an effective and efficient way to make a webpage more noticeable and stand out against others. The favicon is a small logo that appears next to the site name in the browser tab. Favicons can be saved in various file formats. The browser will read the file and anchor the favicon to the favorites list or bookmarks. Different devices will display favicons differently. On a phone or mobile device, the favicon may be displayed as an app logo.

JavaScript

There are 6 main data types in JavaScript. Those data types are string, number, boolean, symbol, object, or the null/undefined keyword. JavaScript variables allow for something called dynamic typing, where the type of a variable can be overwritten by assigning it to a value of a different data type. Objects are non-primitive data types in JavaScript that allow for the storage of key-value pairs, where the keys are referred to as properties. Objects are used in JavaScript by either reading values from the object's properties, updating values, or deleting values. An array in JavaScript is made by making an object without property names, JavaScript will automatically use the strings 0, 1, 2,... as the property names. Arrays are used by accessing values within them with square brackets "[]". JSON is a format for working with data in various applications and objects in various languages. JSON format can be utilized to map an object in a programming language to a string in JSON. Conditionals and loops are used to manipulate the flow of the program in JavaScript. Some examples include using a for in loop to iterate through a JavaScript object, or checking to make sure a certain value within an array matches user input. Object-oriented programming describes utilizing the class and new keywords to define a class prototype and then make an instance of an object respectively. Classes can have added functionality through the use of methods that can be added to a class definition. In JavaScript functions are treated as first class values, which means they can be assigned to variables, can be defined to receive other functions as arguments, and can be defined to return functions as well. Functional programming includes the use of higher order functions, which are functions that recieve functions are arguments.
